# 学科资料下载与论坛交流系统建库语句

本文档按照 E-R 图转换后的关系模式编写。设计原则是：一对多关系不单独建表，直接在“多”的一方保存外键；多对多关系才使用独立关系表。

## 一、关系模式

### 1. 用户表 USERS

`USERS(uid, username, password_hash, created_at, updated_at, avatar_path, user_level, last_ip, is_active, is_admin, points)`

主码：`uid`

候选码：`username`

说明：`points` 保存用户总积分。每成功上传 1 次获得 1 积分，每下载 1 次获得 0.1 积分。

### 2. 文件表 FILES

`FILES(id, file_path, original_name, extension, file_size, description, category, sub_category, created_at, download_count, status, uploader)`

主码：`id`

外键：`uploader` 对应 `USERS(username)`

说明：用户和文件是一对多关系，一个用户可以上传多个文件，所以外键放在文件表中。

### 3. 下载日志表 DOWNLOAD_LOG

`DOWNLOAD_LOG(id, user_id, file_id, file_size, downloaded_at, event_id, source_node, cloud_synced_at)`

主码：`id`

外键：`user_id` 对应 `USERS(uid)`，`file_id` 对应 `FILES(id)`

说明：用户、文件和下载日志都是一对多关系，一条下载日志只对应一个用户和一个文件。

### 4. 帖子表 FORUM_POSTS

`FORUM_POSTS(id, user_id, username, section, title, content, view_count, is_pinned, ip_address, created_at)`

主码：`id`

外键：`user_id` 对应 `USERS(uid)`

说明：用户和帖子是一对多关系，一个用户可以发布多个帖子。

### 5. 评论表 FORUM_COMMENTS

`FORUM_COMMENTS(id, post_id, user_id, username, content, ip_address, created_at)`

主码：`id`

外键：`post_id` 对应 `FORUM_POSTS(id)`，`user_id` 对应 `USERS(uid)`

说明：帖子和评论是一对多关系，用户和评论也是一对多关系，所以评论表同时保存帖子外键和用户外键。

### 6. 论坛图片表 FORUM_IMAGES

`FORUM_IMAGES(id, user_id, username, file_path, original_name, mime_type, file_size, created_at)`

主码：`id`

外键：`user_id` 对应 `USERS(uid)`

说明：用户和论坛图片是一对多关系，一个用户可以上传多张论坛图片。

### 7. 审计日志表 SITE_AUDIT_LOGS

`SITE_AUDIT_LOGS(id, event_type, user_id, username, ip_address, title, content_snippet, created_at)`

主码：`id`

说明：审计日志用于保留历史记录，即使用户信息后续变化，日志也应保留当时记录，因此不设置强制外键。

### 8. 敏感用户记录表 SENSITIVE_USERS

`SENSITIVE_USERS(id, user_id, username, matched_words, source_type, ip_address, created_at)`

主码：`id`

说明：敏感用户记录属于审计类历史数据，不设置强制外键。

### 9. 内容管理员组表 CONTENT_ADMIN_GROUPS

`CONTENT_ADMIN_GROUPS(id, group_name, log_categories, album_categories, user_groups, can_modify_user, can_enter_user_backend, can_modify_user_group, can_manage_user_template, can_publish_site_notice, can_publish_notification, created_at, updated_at)`

主码：`id`

候选码：`group_name`

### 10. 内容管理员成员表 CONTENT_ADMIN_MEMBERS

`CONTENT_ADMIN_MEMBERS(user_id, group_id, created_at)`

主码：`user_id`

外键：`user_id` 对应 `USERS(uid)`，`group_id` 对应 `CONTENT_ADMIN_GROUPS(id)`

说明：该表表示用户加入内容管理员组。由于 `user_id` 是主码，所以一个用户最多属于一个管理员组，一个管理员组可以包含多个用户。独立保存该关系，可以记录用户加入管理员组的时间。

## 二、建库语句

```sql
CREATE DATABASE IF NOT EXISTS download_site
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

USE download_site;
```

## 三、建表语句

```sql
-- 1. 用户表
CREATE TABLE users (
  uid VARCHAR(6) NOT NULL COMMENT '用户编号',
  username VARCHAR(64) NOT NULL COMMENT '用户名',
  password_hash VARCHAR(255) NOT NULL COMMENT '密码哈希',
  created_at DATETIME NOT NULL COMMENT '创建时间',
  updated_at DATETIME NOT NULL COMMENT '更新时间',
  avatar_path VARCHAR(255) DEFAULT '' COMMENT '头像路径',
  user_level VARCHAR(32) DEFAULT 'auto' COMMENT '用户等级',
  last_ip VARCHAR(64) DEFAULT '' COMMENT '最后登录 IP',
  is_active TINYINT NOT NULL DEFAULT 1 COMMENT '是否启用',
  is_admin TINYINT NOT NULL DEFAULT 0 COMMENT '是否管理员',
  points DECIMAL(10,1) NOT NULL DEFAULT 0.0 COMMENT '用户总积分',
  PRIMARY KEY (uid),
  UNIQUE KEY uk_users_username (username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 2. 文件表
CREATE TABLE files (
  id VARCHAR(64) NOT NULL COMMENT '文件编号',
  file_path VARCHAR(1024) NOT NULL COMMENT '文件路径',
  original_name VARCHAR(512) NOT NULL COMMENT '原始文件名',
  extension VARCHAR(32) NOT NULL COMMENT '扩展名',
  file_size BIGINT DEFAULT 0 COMMENT '文件大小',
  description TEXT COMMENT '文件描述',
  category VARCHAR(255) DEFAULT '' COMMENT '一级分类',
  sub_category VARCHAR(512) DEFAULT '' COMMENT '二级分类',
  created_at DATETIME NOT NULL COMMENT '创建时间',
  download_count BIGINT DEFAULT 0 COMMENT '下载次数',
  status VARCHAR(32) DEFAULT 'pending' COMMENT '审核状态',
  uploader VARCHAR(64) NOT NULL COMMENT '上传者用户名',
  PRIMARY KEY (id),
  KEY idx_files_uploader (uploader),
  KEY idx_files_status (status),
  CONSTRAINT fk_files_uploader
    FOREIGN KEY (uploader) REFERENCES users(username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 3. 下载日志表
CREATE TABLE download_log (
  id BIGINT NOT NULL AUTO_INCREMENT COMMENT '日志编号',
  user_id VARCHAR(6) NOT NULL COMMENT '用户编号',
  file_id VARCHAR(64) NOT NULL COMMENT '文件编号',
  file_size BIGINT DEFAULT 0 COMMENT '下载时文件大小',
  downloaded_at DATETIME NOT NULL COMMENT '下载时间',
  event_id VARCHAR(128) DEFAULT '' COMMENT '事件编号',
  source_node VARCHAR(64) DEFAULT '' COMMENT '来源节点',
  cloud_synced_at DATETIME NULL COMMENT '云同步时间',
  PRIMARY KEY (id),
  KEY idx_download_user (user_id),
  KEY idx_download_file (file_id),
  CONSTRAINT fk_download_user
    FOREIGN KEY (user_id) REFERENCES users(uid),
  CONSTRAINT fk_download_file
    FOREIGN KEY (file_id) REFERENCES files(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 4. 帖子表
CREATE TABLE forum_posts (
  id BIGINT NOT NULL AUTO_INCREMENT COMMENT '帖子编号',
  user_id VARCHAR(6) NOT NULL COMMENT '发帖用户编号',
  username VARCHAR(64) NOT NULL COMMENT '发帖用户名',
  section VARCHAR(32) DEFAULT '灌水区' COMMENT '论坛分区',
  title VARCHAR(160) DEFAULT '' COMMENT '标题',
  content TEXT NOT NULL COMMENT '正文',
  view_count BIGINT DEFAULT 0 COMMENT '浏览次数',
  is_pinned TINYINT DEFAULT 0 COMMENT '是否置顶',
  ip_address VARCHAR(64) DEFAULT '' COMMENT '发帖 IP',
  created_at DATETIME NOT NULL COMMENT '创建时间',
  PRIMARY KEY (id),
  KEY idx_posts_user (user_id),
  CONSTRAINT fk_posts_user
    FOREIGN KEY (user_id) REFERENCES users(uid)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 5. 评论表
CREATE TABLE forum_comments (
  id BIGINT NOT NULL AUTO_INCREMENT COMMENT '评论编号',
  post_id BIGINT NOT NULL COMMENT '帖子编号',
  user_id VARCHAR(6) NOT NULL COMMENT '评论用户编号',
  username VARCHAR(64) NOT NULL COMMENT '评论用户名',
  content TEXT NOT NULL COMMENT '评论内容',
  ip_address VARCHAR(64) DEFAULT '' COMMENT '评论 IP',
  created_at DATETIME NOT NULL COMMENT '创建时间',
  PRIMARY KEY (id),
  KEY idx_comments_post (post_id),
  KEY idx_comments_user (user_id),
  CONSTRAINT fk_comments_post
    FOREIGN KEY (post_id) REFERENCES forum_posts(id)
    ON DELETE CASCADE,
  CONSTRAINT fk_comments_user
    FOREIGN KEY (user_id) REFERENCES users(uid)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 6. 论坛图片表
CREATE TABLE forum_images (
  id VARCHAR(64) NOT NULL COMMENT '图片编号',
  user_id VARCHAR(6) NOT NULL COMMENT '上传用户编号',
  username VARCHAR(64) NOT NULL COMMENT '上传用户名',
  file_path VARCHAR(1024) NOT NULL COMMENT '图片路径',
  original_name VARCHAR(512) NOT NULL COMMENT '原始图片名',
  mime_type VARCHAR(64) NOT NULL COMMENT 'MIME 类型',
  file_size BIGINT DEFAULT 0 COMMENT '图片大小',
  created_at DATETIME NOT NULL COMMENT '创建时间',
  PRIMARY KEY (id),
  KEY idx_images_user (user_id),
  CONSTRAINT fk_images_user
    FOREIGN KEY (user_id) REFERENCES users(uid)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 7. 审计日志表
CREATE TABLE site_audit_logs (
  id BIGINT NOT NULL AUTO_INCREMENT COMMENT '审计日志编号',
  event_type VARCHAR(64) NOT NULL COMMENT '事件类型',
  user_id VARCHAR(6) DEFAULT NULL COMMENT '用户编号',
  username VARCHAR(64) DEFAULT '' COMMENT '用户名',
  ip_address VARCHAR(64) DEFAULT '' COMMENT 'IP 地址',
  title VARCHAR(255) DEFAULT '' COMMENT '标题',
  content_snippet TEXT COMMENT '内容摘要',
  created_at DATETIME NOT NULL COMMENT '创建时间',
  PRIMARY KEY (id),
  KEY idx_audit_user (user_id),
  KEY idx_audit_event (event_type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 8. 敏感用户记录表
CREATE TABLE sensitive_users (
  id BIGINT NOT NULL AUTO_INCREMENT COMMENT '记录编号',
  user_id VARCHAR(6) DEFAULT NULL COMMENT '用户编号',
  username VARCHAR(64) DEFAULT '' COMMENT '用户名',
  matched_words VARCHAR(255) DEFAULT '' COMMENT '命中词',
  source_type VARCHAR(64) DEFAULT '' COMMENT '来源类型',
  ip_address VARCHAR(64) DEFAULT '' COMMENT 'IP 地址',
  created_at DATETIME NOT NULL COMMENT '创建时间',
  PRIMARY KEY (id),
  KEY idx_sensitive_user (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 9. 内容管理员组表
CREATE TABLE content_admin_groups (
  id BIGINT NOT NULL AUTO_INCREMENT COMMENT '管理员组编号',
  group_name VARCHAR(64) NOT NULL COMMENT '管理员组名',
  log_categories TEXT COMMENT '日志分类权限',
  album_categories TEXT COMMENT '相册分类权限',
  user_groups TEXT COMMENT '用户组权限',
  can_modify_user TINYINT DEFAULT 0 COMMENT '能否修改用户',
  can_enter_user_backend TINYINT DEFAULT 0 COMMENT '能否进入用户后台',
  can_modify_user_group TINYINT DEFAULT 0 COMMENT '能否修改用户组',
  can_manage_user_template TINYINT DEFAULT 0 COMMENT '能否管理用户模板',
  can_publish_site_notice TINYINT DEFAULT 0 COMMENT '能否发布站点公告',
  can_publish_notification TINYINT DEFAULT 0 COMMENT '能否发布通知',
  created_at DATETIME NOT NULL COMMENT '创建时间',
  updated_at DATETIME NOT NULL COMMENT '更新时间',
  PRIMARY KEY (id),
  UNIQUE KEY uk_admin_group_name (group_name)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 10. 内容管理员成员关系表
CREATE TABLE content_admin_members (
  user_id VARCHAR(6) NOT NULL COMMENT '用户编号',
  group_id BIGINT NOT NULL COMMENT '管理员组编号',
  created_at DATETIME NOT NULL COMMENT '加入时间',
  PRIMARY KEY (user_id),
  KEY idx_admin_members_group (group_id),
  CONSTRAINT fk_admin_members_user
    FOREIGN KEY (user_id) REFERENCES users(uid)
    ON DELETE CASCADE,
  CONSTRAINT fk_admin_members_group
    FOREIGN KEY (group_id) REFERENCES content_admin_groups(id)
    ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 四、规范化说明

1. 用户和文件、用户和帖子、帖子和评论等一对多关系，均在“多”的一方设置外键，不额外建立关系表。
2. 内容管理员成员表用于连接用户和管理员组。当前以 `user_id` 为主码，表示一个用户最多属于一个管理员组；若以后允许一个用户属于多个组，应改用联合主码 `(user_id, group_id)`。
3. `username` 在帖子、评论、图片、审计记录中作为显示快照保存，真正判断用户身份仍以 `user_id` 外键为准。
4. 审计日志和敏感用户记录保留历史操作痕迹，不设置强制外键，防止用户资料变化影响历史记录。
5. 为保持结构简单，管理员组的分类权限使用 `TEXT` 保存。若需要严格拆分权限项，可再建立“管理员组权限关系表”，并使用 `(group_id, permission_id)` 作为联合主码。
